ЦС ЕРЗ - центральный сегмент единого регистра застрахованных лиц. Он содержит информацию о страховании лиц на территории Российской Федерации

Данный сервис позволяет отправлять запросы к ЦС ЕРЗ, с целью получения информации о страховании пациента

Порядок создание запроса следующий:
  1. Создание запроса
  2. Отправка сообщения в ЦС ЕРЗ
  3. Получение ответа от ЦС ЕРЗ

Общая структура интерфейса


Главное меню
  1. Инструкция пользователя - кнопка для отображения данной инструкции
  2. Статус сервиса приема - за отправку сообщений в ЦС ЕРЗ от данного сайта занимается другой сервис. Данное окно показывает текущее состояние этого сервиса. Если сервис не доступен, то вы можете создавать списки, однако отправка произойдет только после активации сервиса. Через меню данного элемента можно повторно запросить статус сервиса
  3. Новый - открывает окно для создания нового запроса
  4. Рабочая область - данное окно содержит список созданных запросов Вашей организации
Обращаем Ваше внимание, что "Рабочая область" отражает данные постранично.
Так же она имеет контекстное меню. Для вызова контекстного меню нажмите правой кнопкой мыши внутри области.

Рабочая область

Рабочая область содержит следующие поля:

  1. Статус отправки - текущие состояние отправки запроса
    Существуют следующие статусы:
    • Новый запрос
    • Запрос в очереди на отправку в ЦС
    • Запрос отправлен в ЦС(ожидание ФЛК)
    • Запрос отправлен в ЦС(ожидание ответа)
    • Ответ получен
    • Ошибка обработки
  2. Результат поиска - результат поиска в ЦС ЕРЗ
    Существуют следующие статусы:
    • Ответ не поступал из ЦС ЕРЗ
    • Пациент найден в ЦС ЕРЗ
    • Пациент не найден в ЦС ЕРЗ
  3. Фамилия
  4. Имя
  5. Отчество
  6. Дата рождения
  7. Полис
  8. Документ
  9. Текущая СМО - последние страхование пациента
  10. Дата - дата создание запроса

Контекстное меню области


Контекстное меню рабочей области
Контекстное меню рабочей области имеет следующие пункты:
  • Просмотр - открыть окно результатов*
  • Новый - открывает окно создания нового запроса
  • Редактировать - открывает окно редактирования запроса(редактирование возможно только запросов со статусом новый)
  • Дублировать - открывает окно создания запроса, с заполненными данными из исходного запроса
  • Установить статус "Новый" - переводит запрос в состояние "Новый"(возможна массовая обработка**)
  • Удалить - удаляет выбранные(возможна массовая обработка**) запросы
  • Обновить - производит обновление данных с сервера
  • Отправить - помечает выбранные(возможна массовая обработка**) списки как готовые к отправке
*Двойной щелчок левой кнопки мыши на строке активирует данный пункт контекстного меню
**Для группового выделения используйте клавиши CTRL и SHIFT

Создание нового запроса

Для создания нового запроса нажмите кнопку "Новый" в рабочем интерфейсе или "Новый" в контекстном меню, после чего откроется окно создания запроса

Окно создания запроса
В данном окне необходимо заполнить данные о пациенте
Обращаем Ваше внимание, что существуют следующие ограничения:
  • Поле "Пол" обязательно к заполнению
  • Поле "Дата рождения" обязательно к заполнению
  • Минимум один из компонентов ФИО должен быть заполнен
  • Минимум один из компонентов ПОЛИС-ДОКУМЕНТ-СНИЛС должен быть заполнен
  • Поле "Номер полиса" обязателен для заполнения, при указании типа полиса
  • Поле "Номер документа" обязателен для заполнения, при указании типа документа

Результат обработки

ЦС ЕРЗ в случае нахождения пациента возвращает следующие данные
  1. Главное ЕНП - ЕНП используемое как текущий в ЦС ЕРЗ
  2. Дата рождения
  3. Дата смерти
  4. Уровень доверия к информации - Уровень доверия к информации, возвращённой в ответ на запрос
    Возможны следующие значения
    • Результату можно доверять - Ответ получен в результате точного совпадения данных
    • Степень доверия высокая - Ответ получен в результате неточного совпадения данных
    • Степень доверия умеренная или низкая - Ответ получен в результате неточного совпадения данных или при неполном сравнении
  5. Ключи поиска - по каким полям удалось найти пациента
    Возможны следующие значения:
    • H01/P01/H02/P02 - [фамилия], [имя], [отчество], [место рождения], [код типа документа, удостоверяющего личность], [номер или серия и номер документа, удостоверяющего личность]
    • H03/P03 - [фамилия], [имя], [отчество], [дата рождения], [СНИЛС]
    • H04/P04 - [фамилия], [имя], [отчество], [дата рождения], [код территории, выдавшей ДПФС], [код типа ДПФС], [серия и номер бланка ДПФС]
    • H05/P05 - [имя], [отчество], [дата рождения], [место рождения], [СНИЛС]
    • H11/P11 - [фамилия], [имя], [отчество], [место рождения], [значение реального идентификатора]
    • H12/P12 - [фамилия], [имя], [отчество], [дата рождения], [значение реального идентификатора]
    • H14/P14 - [фамилия], [имя], [отчество], [дата рождения], [серия и номер бланка ДПФС]
    • H15/P15/H16/P16 - [имя], [отчество], [дата рождения], [СНИЛС]
    • H17/P17/H18/P18 - [фамилия], [имя], [отчество], [место рождения]
  6. История страхования - данные о страховых компаниях пациента со сроком действия страховок
